151 Crochet Collection

A personal cataloguing tool for crochet makers working through the original 151. Sign in with Google, log each finished make with its pattern, yarn, and colors, and watch a private grid fill in toward 151.

Platform
Responsive web · 360–1440px
Auth
Google sign-in only
Model
Personal collection
Themes
Light + dark

01 Overview & goals

The product turns "I'm slowly crocheting all 151" into a tracked, browsable collection. It is a maker's logbook first and a checklist second.

Primary goals

  • Let a maker see, at a glance, how far through the 151 they are.
  • Capture the craft detail that matters to makers — pattern, yarn, hook, colors, time, notes.
  • Make logging a make fast enough to do straight off the hook.
  • Feel like a gallery, not a spreadsheet.

Non-goals (v1)

No social feed, comments, or following. No marketplace or pattern sales. No public profiles — collections are private to the signed-in user. These keep scope tight and the experience calm.

02 Audience

P1The completionist maker

Intermediate+ crocheter steadily working the full 151. Wants progress visibility and a tidy record of what each make took. Visits often, in short bursts.

P2The occasional gifter

Makes a few favourites for friends. Wants to log quickly and look back at which yarn/colors they used so they can remake one. Visits rarely.

03 Information architecture

Flat and shallow — three primary destinations. Everything hangs off the collection grid.

/  Sign in unauthenticated entry
/collection  Collection grid home · authenticated
/c/:dex  Creature detail made or to-make
/c/:dex/edit  Log / edit a make form · modal or page
/account  Account & preferences theme · sign out · data

One canonical grid. Search, type filters, and the made / to-make segmented control are all states of the collection grid — not separate pages. Deep links preserve the active filter in the URL.

04 Critical user journeys

The journeys the design must make effortless, in priority order.

CUJ-1First run → first make

A new visitor signs in and immediately understands the goal: build the 151. They land on an empty grid that invites the first make.

Success: reaches the grid & opens "Log a make" within 2 taps of sign-in.
CUJ-2Log a finished make

The core loop. From the grid, the maker records a creature as made — adding photos, pattern, yarn, colors, and notes — and sees it light up in the grid with updated progress.

Success: a make is saved with at least one photo; grid & progress ring update instantly.
CUJ-3Find & revisit a make

The maker looks up a creature — by name, number, type, or yarn — to view the detail page and remember how they made it.

Success: any of the 151 reachable in ≤ 2 interactions from the grid.
CUJ-4Track progress over time

The maker glances at how far through they are and what's left, optionally narrowing to a single type to plan the next make.

Success: count made / remaining is visible without scrolling on every screen.

05 Flow · Log a make

The primary flow (CUJ-2). Reachable from the persistent "Log a make" button, from any to-make grid cell, or from a creature's detail page.

START
Tap "Log a make"
FAB on grid, or "Make this one" on a to-make cell.
STEP 1
Pick the creature
Searchable list of the 151; pre-filled if entered from a cell.
STEP 2
Add photos
At least one; drag to reorder, set the cover shot.
STEP 3
Pattern & materials
Pattern, designer, upload the pattern PDF, yarn(s), hook, colors, time. All optional but encouraged.
SAVE
Mark as made
Cell fills, progress ring animates, returns to grid.

Rules

  • Only the creature + one photo are required to save; everything else can be filled later.
  • Form autosaves a draft; closing mid-way never loses input.
  • A creature can only be "made" once in a personal collection; re-opening edits the existing make.
  • Color swatches are pickable from a palette or sampled from an uploaded photo.
  • Pattern files accept PDF (one or more — pattern, chart, notes) up to 25 MB each; drag-and-drop or browse, with per-file upload progress and a private-by-default badge. An online pattern URL is an alternative.

06 Flow · Sign in

Single-provider auth keeps the gate frictionless.

START
Landing
Value prop + "Continue with Google".
STEP 1
Google consent
OAuth popup; name, email, avatar only.
STEP 2
Provision
First time: create empty 151. Returning: load collection.
DONE
Collection grid
Lands on home, signed in, theme preference restored.

Failure handling. If OAuth is cancelled or errors, return to landing with a non-blocking message ("Sign-in didn't complete — try again"). Never trap the user in a blank state.

08 States & edge cases

Every data surface must define these. Empty and loading states are part of the design, not afterthoughts.

Empty collection

0 made. Grid shows all 151 as to-make; hero invites the first log with an arrow to the FAB.

Loading

Skeleton cells in dex order; header progress shows a muted placeholder, never 0/151 flashing.

No search results

"No matches for 'x'." Offer to clear filters; keep search field focused.

Complete · 151/151

Celebration state on the grid + a subtle persistent banner. Goal reached.

Draft in progress

A started-but-unfinished make. Shows in the grid with an amber "Draft" badge and a WIP-textured tile; tapping resumes the log form prefilled. Doesn't count toward 26 / 151.

Photo upload failing

Per-photo retry; the make still saves with the photos that succeeded.

Offline

Read cached collection; queue edits and sync when back. Non-blocking banner.

09 Responsive behavior

Mobile-first. The grid is the spine; column count flexes with width.

BreakpointBehavior
Mobile · 360–6402-col grid; search collapses into header icon → full-width on focus; FAB bottom-right; detail is single column with gallery on top.
Tablet · 641–10244–5 col grid; progress card sits beside the hero; detail becomes two columns.
Desktop · 1025–14406–8 col grid; sticky gallery on detail; full filter bar inline; max content width 1240px.
  • Touch targets ≥ 44px; grid cells remain tappable at the smallest size.
  • No horizontal scroll at any width; cells reflow, never shrink below legibility.

10 Accessibility

  • Made / to-make never relies on color alone — a check icon + label + dashed border carry the state too.
  • Text contrast ≥ 4.5:1 in both themes; the soft-violet accent is reserved for ≥ 3:1 UI, not body text.
  • Full keyboard path: "/" to search, arrow-navigate the grid, Enter to open, Esc to close overlays.
  • Visible focus ring on every interactive element; respects prefers-reduced-motion.
  • Every photo slot has alt text; grid cells announce "Bulbasaur, №001, made / to make".

11 Auth, data & privacy

Google is the sole identity provider. The app requests the minimum scope — name, email, profile photo — and stores only what the maker enters about their makes. Collections are private by default; there is no public surface in v1.

  • Clear sign-out and "delete my collection" in Account.
  • Export collection (JSON / printable PDF) so the maker owns their log.
  • Independent fan project — disclaimer of non-affiliation in the footer.
